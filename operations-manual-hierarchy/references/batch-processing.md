## 【模式二：文档批量解析存储】

适用于用户传入完整文档（Markdown、TXT、JSON 等），需自动解析并将不同内容分发存入对应分类。

### 第一步：内容切块

读取传入文档，按以下规则将内容切割为独立的「条目单元」：

**拆分规则：**

| 切块依据 | 说明 |
|---------|------|
| 章节标题（`#` / `##` / `###`） | 每个有实质内容的章节作为候选条目单元 |
| 语义独立性 | 相互独立、无强依赖关系的内容块分别存储 |
| 内容类型差异 | 不同类型的内容（如角色定义、操作流程、配置参数）拆分存储 |
| 长度上限 | 单个条目单元原则上不超过 800 字，超出时按语义边界继续拆分 |

**合并规则（避免碎片化）：**

| 合并条件 | 说明 |
|---------|------|
| 内容过短 | 单独章节内容不足 50 字，且语义上属于相邻章节的补充，则与相邻单元合并 |
| 上下文强依赖 | 内容无法脱离上下文独立理解（如某操作步骤的子步骤），整体保留为一个单元 |
| 同一概念分散描述 | 同一知识点散落在多个小节中，合并为一个完整的条目单元 |

> ⚠️ **拆分与合并的优先级**：语义完整性 > 字数限制。宁可一个单元稍长，也不强行拆散完整的逻辑。

### 第二步：分类推断

对每个条目单元，依据**内容语义**推断一级分类和二级分类。

**推断优先级（依次执行，前者优先）：**

1. **用户显式指定**：若用户在上传时指定了分类，直接采用，无需推断

2. **复用已有分类**：读取一级索引，查找语义匹配的一级分类和二级分类，优先复用已有分类，保持分类体系稳定

3. **基于内容语义自由推断**：不限于预设类型，根据文档内容自主推断最合适的一级分类和二级分类名称。以下为常见场景的参考：

   | 内容特征 | 推断一级分类（参考） | 推断二级分类（参考） |
   |---------|-----------|-----------------|
   | Agent 角色定义、人设、行为准则 | Agent配置 | 角色定义 |
   | 工具列表、工具调用规则 | Agent配置 | 工具配置 |
   | 操作流程、SOP、步骤说明 | Agent配置 | 操作流程 |
   | 目录结构、文件规范、命名规则 | Agent配置 | 规范约束 |
   | 数据格式、模板、示例 | Agent配置 | 数据格式 |
   | 注意事项、限制条件、禁止行为 | Agent配置 | 约束规则 |
   | 技术实现、代码片段、算法 | 技术学习 | 具体技术领域 |
   | 项目背景、产品信息 | 项目知识 | 具体项目名 |
   | 个人偏好、用户习惯 | 个人信息 | 具体类别 |

   > 以上表格仅为参考示例，**不得机械套用**。若文档内容属于表格中未覆盖的领域（如医疗、法律、财务等），应根据实际内容自由创建更准确的分类名称。

4. **歧义处理**：若某条目单元同时符合多个分类，选最主要的分类存储，并在 `tags` 中注明其他相关分类关键词，同时在 `related` 字段关联语义相近的已有条目

### 第三步：展示存储计划并等待确认

在实际写入前，向用户展示完整解析结果，**等待用户确认后再执行写入**。

根据条目单元数量，采用不同展示方式：

**≤ 10 条时，展示详细计划：**

```
📋 文档解析完成：<文档名>
共识别 N 个条目单元，存储计划如下：

序号 | 条目标题             | 一级分类     | 二级分类   | 重要度
-----|---------------------|-------------|-----------|-------
  1  | Agent角色定义        | Agent配置    | 角色定义   | ⭐⭐⭐⭐⭐
  2  | 目录结构规范          | Agent配置    | 规范约束   | ⭐⭐⭐⭐
  3  | Python异步编程示例    | 技术学习     | Python    | ⭐⭐⭐
 ... | ...                 | ...         | ...       | ...

是否确认按以上计划存储？如需调整某条的分类或标题，请指出序号后我再写入。
```

**> 10 条时，展示分组摘要，减少认知负担：**

```
📋 文档解析完成：<文档名>
共识别 N 个条目单元，按分类汇总如下：

📁 Agent配置（共 X 条）
  └─ 角色定义：Agent角色定义、...
  └─ 规范约束：目录结构规范、命名规则、...

📁 技术学习（共 Y 条）
  └─ Python：异步编程示例、...

📁 <其他分类>（共 Z 条）
  └─ ...

💡 如需查看某分类的详细列表，请回复「展开 <分类名>」
是否确认按以上计划存储？或指出需要调整的部分？
```
