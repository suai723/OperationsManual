## 错误处理

| 异常情况 | 处理方式 |
|---------|---------|
| 随机 ID 与现有目录冲突 | 重新生成 ID，最多重试 5 次，仍冲突则报错提示用户 |
| 文档解析后无有效内容块 | 提示用户文档内容无法解析，并说明原因（内容过短/格式无法识别），请求用户手动指定内容 |
| 子 Agent 写入超时（>300秒未返回回报文件，仅并行路径） | 主 Agent 输出已完成分类的写入结果，提示用户按 [维护操作](maintenance.md) 重建索引 |
| 回报文件中包含错误条目 | 主 Agent 在输出结果中单独列出错误条目，提示用户手动重试或检查 |
| 写入过程中断（无回报文件） | 输出已成功写入的文件列表，提示用户按 [维护操作](maintenance.md) 重建索引；**清理残留的 `.dispatch_*` 和 `.result_*` 文件**；下次写入前优先检查上次中断点 |
| 内容与已有条目高度相似 | 批量模式下自动跳过并记录，单条模式下按「去重规则」中定义的交互流程处理 |
| 分类推断置信度低 | 在计划展示阶段标注 `⚠️ 分类待确认`，提示用户复核该条目的分类是否合适 |
| 索引文件损坏或格式异常 | 提示用户按 [维护操作](maintenance.md) 由 Agent 重建各级 index.md，不自动覆盖已有索引 |
| 分发文件写入失败 | 中止本次批量写入，提示用户检查目录写入权限 |
| 单一级分类场景下二级分类数为 1 | 降级为主 Agent 直接串行写入，无需启动子 Agent 或分片（仅一个分片时并行无意义） |
