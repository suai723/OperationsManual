# 工作手册根目录与 {WORKSPACE} 占位符迭代

## 路径约定（采纳写法）

- 目录名使用 `**operations_manual**`（全小写、下划线）。你消息中的 `operatIons_manual` 视为笔误，按常规路径名实现。
- `**{WORKSPACE}**`：表示 Agent 当前**工作区根目录**（与 Cursor 工作区、OpenClaw workspace 根等价的抽象根）。所有原 `/home/node/.openclaw/workspace/...` 及类似硬编码，在**技能 Markdown 文档**中一律改为 `{WORKSPACE}/...`。
- 分级库根目录规范为：`**{WORKSPACE}/operations_manual/hierarchy/`**（其下仍为 `index.md`、`category_`*、`subcategory_*`、`entry_*`、`.dispatch_*` 等，结构不变）。
- 在 [references/directory-structure.md](e:\Myproject\OperationsManual\operations-manual-hierarchy\references\directory-structure.md) 或 [references/operation-modes.md](e:\Myproject\OperationsManual\operations-manual-hierarchy\references\operation-modes.md) 顶部增加**简短说明块**：`{WORKSPACE}` 含义、命令行/Bash 中需替换为实际路径或先 `export WORKSPACE=...` 再使用 `$WORKSPACE/...`（因 shell 不会自动展开字面量 `{WORKSPACE}`）。

## 影子摘要路径

- 当前文档为 `memory/workbook_shadow_index.md`。与工作手册根一并统一为 `**{WORKSPACE}/memory/operations_manual_shadow_index.md`**（文件名与目录命名一致，避免继续带 `workbook`）。
- 若需兼容已存在的 `workbook_shadow_index.md`，可在 [references/maintenance.md](e:\Myproject\OperationsManual\operations-manual-hierarchy\references\maintenance.md) 或说明块中加一句「存量可手工合并或迁移」；**默认不**在脚本里自动双写，保持技能文档单一真源。
