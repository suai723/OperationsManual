# 检索策略详述

本文件为 operations-manual-reader 技能的完整检索流程参考。SKILL.md 中保留精简版与指针。

## 总体架构

```
意图分流 → (直读 | 浏览索引 | 多路并行探查) → 合并排序 → 降级链 → 回答或澄清
```

全程维护**已读台账**：任一 Read 前先查已读集合，已完整读过的文件不再重复读取。

## 意图分流

| 用户意图 | 路径 |
|---------|------|
| 明确 entry ID 或相对路径 | glob 定位 → Read；文件不存在则转并行探查 |
| 浏览结构 | 从根 `index.md` 起渐进读取各级 `index.md` |
| 模糊 / 自然语言 / 标题不完整 | 进入**多路并行探查** |

## 多路并行探查

### 脚本执行

通过 Bash 单次调用 `scripts/parallel_retrieve.py`：

```bash
python scripts/parallel_retrieve.py --root "$WORKSPACE/operations_manual/hierarchy" --query "用户查询"
```

脚本内以 `concurrent.futures.ThreadPoolExecutor` 将**索引路、元数据路、全文路**三路真并行执行，输出 stdout JSON。禁止默认串行实现。

### 四路详述

#### 索引路

在库根、`category_*`、`subcategory_*` 的 `index.md` 中对用户关键词 grep。命中行若指向某目录或条目，从该**锚点**继续向下渐进读取（下一层 `index.md` 或直读 entry），不盲目打开无关文件。

#### 元数据路

扫描 `entry_*.md` 的 YAML frontmatter（`title` / `tags` / `summary`）。

**无 YAML 兼容**：若文件没有 `---` 块或解析失败，该文件记为 `yaml_absent: true`，该路仍正常结束（空字段，不抛错），合并时降权但不排除该条目——全文路、索引路仍可贡献命中。

#### 全文路

与索引、元数据**同时**对限定 glob 下 `entry_*.md` 正文做受控 grep（范围可与索引缩小联动）。不依赖「等前两路结束再启动」。

#### 向量路（可选）

**仅当**宿主提供 `memory_search`（或等价向量检索）时由 Agent 在调脚本的**同一轮**并行发起。不支持向量时整路跳过，不等待、不阻塞脚本三路、不虚构结果。

检测方式：以宿主实际 `allowed-tools` 是否含向量工具为准。

### 与向量路的衔接

脚本启动后 Agent 立即并行调 `memory_search`（若可用）；合并时将脚本 JSON 与向量返回无序融合（谁先到谁先入池，最后统一排序）。

## 合并规则

多路结果按以下优先级去重排序：

1. 直读路径已存在的结果
2. 索引表中出现且与 query 词面重合度高的条目
3. 向量分数（仅该路存在时）
4. `yaml_absent` 条目仅用其它路信号参与排序（忽略 summary）

最终展示 **Top N（3～8 条）**，并说明各来自哪条路。

## 降级链

并行仍无高置信结果时，按顺序收紧：

### 1. 放宽词面

同义词、去后缀、英文/缩写变体后再跑一轮脚本（遵守已读台账）。

### 2. 全局渐进式兜底

**触发条件**：
- **有向量路**：索引关键词路与向量路均未达较高置信度
- **无向量路**：不参与「向量低」判断，仅当关键词路（及合并置信）仍低于较高阈值时触发

**执行**：从库根 `operations_manual/hierarchy/index.md` 开始，按**根 → 各 category → 其下 subcategory → 必要时再 entry** 的顺序系统性渐进读取，补全并行「点状 grep」可能漏掉的全局结构信息。每读完一层再决定是否深入下一层。跳过已在已读集合中的文件。

### 3. 澄清

列出一级分类名称与 ID，请用户点选或补充领域词。

### 4. 如实未命中

向用户如实反馈并附当前库内主要分类供下一轮澄清。禁止在无写入意图时代为修改索引。

## 已读台账

- **范围**：单次用户查询/任务闭环内，对每次完整 Read 的文件路径记入已读集合
- **规则**：已完整读过的 `index.md` / `entry_*.md` 不再重复读取；需引用片段时依赖缓存内容
- **grep 与 Read**：可先 grep 定位行再首次 Read 必要区间；若全文已在集合中则不再 Read

## 渐进式读取的三类场景

1. **主动浏览**：用户只想看结构 → 从根 `index.md` 起按需进入各级 `index.md`
2. **索引命中锚点钻取**：grep 命中行对应某目录或条目 → 从锚点向下渐进
3. **全局渐进兜底**：并行双低时从根 index 系统性渐进

## 与 hierarchy 技能的边界

- Reader 的并行仅限**只读检索**，不与 hierarchy 的写入分片混用分发文件
- 默认以脚本真并行为准；仅在宿主明确禁止多线程的极少数环境中声明顺序降级（索引 → 元数据 → 全文 → 可选向量），此为**例外**非默认

## 空库 / 库未初始化场景

若 `{WORKSPACE}/operations_manual/hierarchy/` 不存在或根 `index.md` 缺失：
- 脚本三路均返回空结果
- 全局渐进在读根 index 时立即发现不存在
- Agent 应如实告知「当前工作区未建立工作手册目录」并提示用户使用 operations-manual-hierarchy 技能导入内容
